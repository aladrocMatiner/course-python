# الملحق B · خوارزميات أساسية: البحث في Python

[English](README.md) · [Español](README.es.md) · [Català](README.ca.md) · [Svenska](README.sv.md) · العربية

## ماذا سنبني؟
سنستكشف ثلاثة أنماط أساسية للبحث: البحث الخطي، والبحث الثنائي، والبحث بالعرض (BFS) في الرسوم البيانية البسيطة. سننفّذ دوال نقية، ونحلل التعقيد الزمني، ونتدرّب على اختبارات تثبت صحة النتائج.

## مسار التعلّم
1. **البحث الخطي**: بسيط لكن تكلفته O(n).
2. **البحث الثنائي**: يتطلب قائمة مرتّبة وتكلفته O(log n).
3. **البحث في البنى المتصلة**: BFS للرسوم البيانية والأشجار.
4. **تحليل التعقيد والحالات الحدّية**.
5. **اختبارات آلية لكل خوارزمية**.

## أهداف التعلّم
- تنفيذ خوارزميات بحث تكرارية ومعرفة مواضع استخدامها.
- فهم أثر حجم المدخلات في تكلفة كل خوارزمية.
- تحديد الشروط المسبقة، مثل ضرورة ترتيب القائمة.
- تطبيق BFS باستخدام طوابير `deque` لاجتياز الرسوم البيانية.
- كتابة اختبارات للنجاح والفشل والمدخلات الفارغة.

## لماذا يهم هذا؟
يمكن اختزال كثير من المشكلات الحقيقية في عبارة «اعثر على شيء». اختيار الخوارزمية المناسبة يمنع العمل غير الضروري ويفتح الباب أمام بنى بيانات أكثر تقدمًا.

### مغامرة صغيرة
قد يشبه البحث في قائمة العثور على كتاب في غرفتك. إذا كانت الغرفة فوضوية تفحص الأشياء واحدًا تلو الآخر، وهذا بحث خطي. وإذا كانت الكتب مرتبة أبجديًا، تقفز إلى المنتصف وتستبعد نصف الاحتمالات بسرعة، وهذا بحث ثنائي. اختيار الاستراتيجية الصحيحة يوفر الوقت.

## 1. البحث الخطي

```python runnable
def busqueda_lineal(elementos, objetivo):
    for indice, valor in enumerate(elementos):
        if valor == objetivo:
            return indice
    return -1
```

- التعقيد O(n)، وهو مناسب للمجموعات الصغيرة أو غير المرتبة.
- إذا وُجدت قيم مكررة، يعيد موضع أول تطابق.

### اختبار سريع
```python illustrative
def test_busqueda_lineal():
    datos = [3, 5, 7]
    assert busqueda_lineal(datos, 5) == 1
    assert busqueda_lineal(datos, 2) == -1
```

## 2. البحث الثنائي

```python runnable
def busqueda_binaria(ordenados, objetivo):
    izquierda, derecha = 0, len(ordenados) - 1
    while izquierda <= derecha:
        medio = (izquierda + derecha) // 2
        valor = ordenados[medio]
        if valor == objetivo:
            return medio
        if valor < objetivo:
            izquierda = medio + 1
        else:
            derecha = medio - 1
    return -1
```

- يتطلب قائمة مرتبة تصاعديًا.
- تكلفته O(log n)، لأن كل تكرار يخفض مساحة البحث إلى النصف.

### حالات ينبغي تغطيتها
- وجود الهدف في البداية والمنتصف والنهاية، وعدم وجوده.
- القائمة الفارغة؛ عندها تكون `derecha = -1` وتعود الدالة فورًا بالقيمة -1.

## 3. البحث بالعرض (BFS)

```python runnable
from collections import deque

def bfs(grafo, inicio, objetivo):
    if inicio not in grafo or objetivo not in grafo:
        return False
    visitados = set([inicio])
    cola = deque([inicio])

    while cola:
        nodo = cola.popleft()
        if nodo == objetivo:
            return True
        for vecino in grafo.get(nodo, []):
            if vecino not in visitados:
                visitados.add(vecino)
                cola.append(vecino)
    return False
```

- `grafo` قاموس تقترن فيه كل عقدة بقائمة من جيرانها.
- التعقيد O(V + E)، حيث V عدد الرؤوس وE عدد الحواف.
- تتحقق هذه النسخة المنطقية من إمكانية الوصول. وتستعيد نسخة تتبع الآباء أقصر مسار، أما اكتشاف الدورات فيحتاج شرطًا آخر.

### مثال على رسم بياني
```python illustrative
grafo = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": [],
    "D": []
}
assert bfs(grafo, "A", "D") is True
assert bfs(grafo, "C", "D") is False
```

---

## 4. مقارنة التعقيد

| الخوارزمية | أفضل حالة | أسوأ حالة | الشروط المسبقة |
| --- | --- | --- | --- |
| البحث الخطي | O(1)، العنصر الأول | O(n) | لا شيء |
| البحث الثنائي | O(1)، العنصر الأوسط | O(log n) | قائمة مرتبة |
| BFS | O(1)، البداية هي الهدف | O(V + E) | تمثيل للرسم البياني |

- تنمو O(log n) أبطأ بكثير من O(n).
- تكون BFS أعلى تكلفة، لكنها تستكشف بنى أغنى.

## أخطاء شائعة
- نسيان شرط الخروج من البحث الثنائي، مما يؤدي إلى حلقة لا نهائية.
- مقارنة قيم من دون توحيد أنواعها، مثل مقارنة نصوص بأعداد صحيحة.
- إعادة استخدام بنى قابلة للتغيير في BFS من دون نسخها، فتتشارك المراجع.
- عدم التحقق من وجود عقدة البداية في الرسم البياني.

---

## حلول مشروحة
1. **القيم المكررة**: يحافظ استخدام مجموعة على O(n)، لأن متوسط تكلفة كل إدراج O(1). أما الحل ذو الحلقتين فتكلفته O(n²) ولا يتوسع جيدًا.
2. **البحث الثنائي**: خزّن كل تطابق وتابع في النصف الأيسر؛ وهكذا تعيد أول ظهور أو -1 عند غياب التطابق.
3. **BFS مع المسار**: خزّن `padres[vecino] = nodo`. وعند العثور على الهدف، أعد بناء المسار بالرجوع من الهدف إلى البداية.
