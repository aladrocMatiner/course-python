## ماذا سنبني؟

قبل أن نكتب أي شيفرة، سنجهّز «ورشة بايثون» الخاصة بك: سنثبّت إصدارًا حديثًا من Python، ونتأكد من أن `pip` يعمل، ثم نتعلم الدورة الكاملة: عدّل ← احفظ ← شغّل ← شخّص ← أصلح. الهدف بسيط: عندما تنفّذ `python --version` يجيبك الحاسوب، وعندما تشغّل أول برنامج محفوظ تعرف أي ملف فتحه Python وكيف تتعافى من أول traceback.

## مسار التعلم

1. **ثبّت Python** وتأكد من أن الأمر يعمل.
2. **افحص `pip`** حتى نستطيع تثبيت المكتبات عند الحاجة.
3. **البيئات الافتراضية**: أنشئ «صندوقًا» صغيرًا (`.venv`) لكل مشروع.
4. **مستوى إضافي**: استخدم `asdf` إذا أردت إعدادًا احترافيًا جدًا.
5. **دورة البرنامج الأول**: أنشئ مجلدًا تملكه، واحفظ `hello.py`، وشغّله، واقرأ خطأً متوقعًا، ثم أصلحه وأعد التشغيل.

## أهداف التعلم

- التحقق من تثبيت Python وفهم معنى رقم الإصدار.
- التأكد من أن `pip` يعمل وتحديثه.
- إنشاء بيئة افتراضية (`venv`) لمشروع وتفعيلها.
- تجنّب مشكلة «يعمل على جهازي فقط» الشائعة.
- التمييز بين المحرر وملف `.py` المحفوظ وshell والمفسّر المختار وstandard output وREPL التفاعلي.
- تشغيل ملف من working directory الصحيح، واستخدام الأجزاء المستقرة من traceback لإصلاح `NameError`.

- أبقِ هذا الفصل مفتوحًا في علامة تبويب أو جهاز آخر كي تستطيع قراءة خطوات الاسترداد إذا فشل أمر ما.
- **مسار الإعداد والبرنامج الأول الأساسي · 60–90 دقيقة:** أكمل الأقسام 1 و2 و4 و5 ونقطتي التحقق. النتيجة: مجلد تعلم محلي مؤقت يحتوي `hello.py` بعد إصلاحه ويطبع سطرًا واحدًا مطابقًا. يمكنك التوقف بعد نقطة تحقق البرنامج الأول والعودة لاحقًا إلى أدوات البيئة الاختيارية.

## 1. تثبيت Python (يُنصح بالإصدار 3.11 أو أحدث)

### Windows

1. افتح [صفحة تنزيل Python الرسمية لنظام Windows](https://www.python.org/downloads/windows/) ونزّل المثبّت الرسمي (`python-3.x.x-amd64.exe`).
2. أثناء التثبيت، فعّل خيار **“Add python.exe to PATH”** حتى لا تضبط المسار يدويًا لاحقًا.
3. أكمل التثبيت ثم تحقّق:
   ```powershell illustrative
   python --version
   pip --version
   ```
4. بديل آلي في Windows 11 أو أحدث:
   ```powershell illustrative
   winget install --id Python.Python.3.11 --source winget
   ```

### macOS

1. نزّل ملف `.pkg` من [صفحة تنزيل Python الرسمية لنظام macOS](https://www.python.org/downloads/macos/).
2. شغّل المثبّت واقبل الخيارات الافتراضية؛ سيُثبَّت Python داخل `/Library/Frameworks/Python.framework`.
3. تحقّق:
   ```bash illustrative
   python3 --version
   pip3 --version
   ```
4. بديل باستخدام Homebrew:
   ```bash illustrative
   brew update
   brew install python@3.11
   python3.11 --version
   python3.11 -m pip --version
   ```
   لا تكتب `/opt/homebrew` بصورة ثابتة في ملف إعداد الطرفية؛ يستخدم Homebrew بادئات مختلفة على Apple Silicon وIntel. إذا لم يُعثر على `python3.11`، فنفّذ `brew info python@3.11` واتبع الملاحظة الخاصة بتثبيتك.

### Linux (Debian/Ubuntu)

```bash illustrative
sudo apt update
sudo apt install -y python3 python3-pip python3-venv
python3 --version
pip3 --version
```

إذا كانت توزيعتك توفر إصدارًا أقدم من 3.11، فلا تضف مستودعًا خارجيًا غير مُراجع لمجرد متابعة الفصل. استخدم تعليمات Python الرسمية لتوزيعتك أو مسار `asdf` الاختياري أدناه.

### Linux (Fedora/CentOS/RHEL)

```bash illustrative
sudo dnf install -y python3 python3-pip
python3 --version
pip3 --version
```

## 2. التأكد من توفر `pip`

- افحص أولًا `pip` المرتبط بالمفسّر الذي ستستخدمه:
  ```bash illustrative
  python3 -m pip --version
  ```
- `ensurepip` وحدة اختيارية في CPython. إذا ثبّت Python من `python.org` وكانت الوحدة موجودة، فيمكنها تهيئة `pip` من دون تنزيله:
  ```bash illustrative
  python3 -m ensurepip --upgrade
  ```
- إذا ظهر `No module named ensurepip`، فاستخدم المثبّت الرسمي أو مدير حزم نظامك (`python3-pip` في عائلتي Debian وFedora)، ولا تنسخ سكربت تهيئة عشوائيًا.
- حدّث `pip` فقط بعد تفعيل البيئة الافتراضية للمشروع:
  ```bash illustrative
  python -m pip install --upgrade pip
  ```

## 3. إدارة أكثر من إصدار باستخدام `asdf` (اختياري واحترافي)

يتيح لك `asdf` تثبيت إصدارات Python مختلفة والتبديل بينها —وبين أدوات أخرى— حسب المشروع، فتظل البيئات متسقة.

هذا المسار اختياري، ولا يحتاج إليه checkpoint الأساسي. ثبّت إصدار `asdf` الحالي باتباع [دليل البدء الرسمي](https://asdf-vm.com/guide/getting-started.html) الذي يغطي نظامك ومعماريته. تستخدم الأوامر الآتية الواجهة التنفيذية الحالية؛ ولا تنطبق تعليمات `asdf.sh` و`asdf global` القديمة.

1. تحقّق من التثبيت:
   ```bash illustrative
   asdf --version
   ```
2. أضف ملحق Python المجتمعي بعد مراجعة المصدر الذي يعرضه فهرس الملحقات الرسمي. إذا ظهر `python` بالفعل في `asdf plugin list` فتجاوز الأمر:
   ```bash illustrative
   asdf plugin add python
   ```
3. ثبّت أحدث إصدار تصحيحي مستقر من 3.11 واكتب الاختيار الفعلي في `.tool-versions` الخاص بالمشروع:
   ```bash illustrative
   asdf install python latest:3.11
   asdf set python latest:3.11
   ```
4. تحقّق:
   ```bash illustrative
   python --version
   python -m pip --version
   ```

قورنت هذه التعليمات الحساسة للإصدارات بوثائق Python وHomebrew و`asdf` الرسمية في 2026-07-13.

الفوائد:

- يستطيع كل مستودع تحديد ملف `.tool-versions` لتثبيت الإصدارات المطلوبة.
- تتجنّب التعارض بين مشاريع تحتاج إلى إصدارات مختلفة.

## 5. من مجلد إلى برنامجك الأول

هذه هي الدورة الصغيرة التي ستكررها في الكتاب كله:

1. يغيّر **المحرر** النص داخل source file، لكنه لا يثبت أن البرنامج يعمل.
2. **احفظ** الملف باسم `hello.py`، وتأكد أن اسمه لا ينتهي بلاحقة ثانية مخفية مثل `.txt`.
3. يشغّل **shell** الأوامر داخل working directory الحالي. افتح الطرفية في مجلد التعلم أو انتقل إليه صراحة.
4. يفتح **مفسّر Python** المختار ملف `hello.py` وينفّذه.
5. يرسل `print` النص إلى **standard output** فتظهر النتيجة في الطرفية.

المحرف `>>>` يدل على **REPL** التفاعلي الذي يستقبل تعبيرات Python واحدًا تلو الآخر. أما `python hello.py` فهو أمر shell؛ اخرج من REPL باستخدام `exit()` قبل كتابته. استبدل `python` في هذا القسم بالصيغة التي نجحت وسجلتها (`python3` أو `py`) عند الحاجة، واستخدم الصيغة نفسها باستمرار.

### جهّز working directory تملكه

استخدم مجلدًا تملكه، لا مجلد تثبيت Python ولا مجلد نظام. الأوامر التالية أمثلة؛ اختر ما يناسب shell لديك.

```bash illustrative
# macOS/Linux: quote a path that contains spaces
mkdir -p "$HOME/Python Course/chapter-01"
cd "$HOME/Python Course/chapter-01"
```

```powershell illustrative
# Windows PowerShell: quotes preserve the spaces
New-Item -ItemType Directory -Force "$HOME\Python Course\chapter-01"
Set-Location "$HOME\Python Course\chapter-01"
```

يتغير prompt عادة بعد `cd` أو `Set-Location`. وإذا استخدمت أمر «Open Folder» في محرر، فافتح المجلد نفسه. لا يلزم تغيير اسم مجلد شخصي لمجرد احتوائه على مسافات.

### توقّع واحفظ وشغّل

قبل التشغيل، توقّع السطر الوحيد الذي سيطبعه الملف. أنشئ في المحرر ملف `hello.py` بالمحتوى التالي تمامًا. تظهر الملاحظة الناجحة المرتبطة بالكتلة القابلة للتشغيل بعدها مباشرة:

```python runnable
print("Hello, Python!")
```
```text output
Hello, Python!
```

احفظه، ثم اكتب الأمر التالي في **shell**، لا داخل الملف ولا بعد `>>>`:

```bash illustrative
python hello.py
```

إذا قال Python إنه لا يستطيع فتح `hello.py`، فالحقيقة المستقرة هي أنه لم يجد المسار المطلوب انطلاقًا من working directory الحالي. افحص المجلد والتهجئة ولا حقة `.py` وهل حفظ المحرر الملف. لا تُعد تثبيت Python، ولا تستخدم صلاحيات إدارية، ولا تنقل التمرين إلى مجلد نظام.

### أول traceback متعمّد والتعافي منه

غيّر `print` فقط إلى `pritn`، ثم احفظ وتوقّع النتيجة. هذا block متوقع الفشل:

<!-- bookcheck: expect-error="NameError" -->
```python expected-error
pritn("Hello, Python!")
```

قد تتغير صياغة traceback باختلاف إصدار Python. اقرأ الدلائل المستقرة من الأسفل إلى الأعلى:

- فئة الاستثناء الأخيرة هي `NameError`؛
- الرسالة الأخيرة تحدد الاسم المجهول `pritn`؛
- مسار source والسطر المبلّغ عنهما يعيدانك إلى الملف المحفوظ.

أصلح التهجئة وحدها، واحفظ مجددًا ثم أعد التشغيل:

```python runnable
print("Hello, Python!")
```

يجب أن ترى `Hello, Python!` من جديد. إعادة التشغيل الناجحة تكمل التعافي؛ قراءة الخطأ وحدها لا تكفي.

## أخطاء شائعة

- تثبيت Python من دون تفعيل “Add python.exe to PATH” في Windows، فيصبح أمر `python` غير معروف.
- استخدام `pip` تابع لإصدار Python مختلف عن الإصدار الذي تشغّله، فتُثبَّت المكتبة في مكان آخر.
- نسيان تفعيل `.venv`، فيستخدم المشروع dependencies غير المقصودة.
- استخدام `sudo pip install`، وقد يفسد حزم النظام؛ استخدم `.venv` بدلًا منه.
- كتابة `python hello.py` بعد محث `>>>`؛ اخرج من REPL باستخدام `exit()` ثم شغّل الأمر في shell.
- التشغيل من مجلد آخر أو حفظ الملف باسم `hello.py.txt`؛ افحص المجلد الحالي واسم الملف الكامل قبل تغيير تثبيت Python.
- التعديل من دون حفظ؛ يشغّل المفسّر المحتوى الأقدم المحفوظ كما ينبغي، لذا احفظ ثم أعد التشغيل.

## حلول مفسرة

- إذا لم يوجد `python`: افحص PATH أو أعد التثبيت في Windows، وجرّب `python3` في macOS وLinux.
- لتجنّب `pip` الخطأ، استخدم دائمًا `python -m pip ...`؛ عندها يرتبط `pip` بنفس Python.
- إذا تعذر تفعيل `venv`، تأكد من أنك في المجلد الصحيح وأن `.venv/` موجود.
- عند ظهور `NameError` بسبب `pritn`: استخدم الملف والسطر المبلّغ عنهما، وصحّح `pritn` إلى `print`، ثم احفظ وأعد التشغيل حتى يظهر السطر الناجح المطابق.
- إذا تعذر على Python فتح `hello.py`: عد إلى مجلد تعلم تملكه وتحقق من اسم الملف الكامل. تحافظ علامات الاقتباس على المسافات في المسار؛ حذف المسافات ليس هو الإصلاح.

## Checkpoint وتقييم ذاتي

### Checkpoint البرنامج الأول

في مجلد تعلم تملكه، أنشئ `hello.py`، وتوقّع ناتجه، واحفظه، ثم شغّله بأمر المفسّر الذي سجلته. بعد ذلك أدخل `pritn`، وحدد ملف المصدر والسطر و`NameError` والاسم الذي أخطأت في كتابته، ثم أصلحه واحفظ وأعد التشغيل. يجب أن يكون standard output النهائي مطابقًا تمامًا لـ `Hello, Python!` يتبعه سطر جديد.

امنح نفسك نقطة لكل معيار:

- **الصحة:** يطبع الملف بعد إصلاحه السطر المتوقع المطابق.
- **الوضوح:** تستطيع التمييز بين المحرر وملف المصدر وأمر shell والمفسّر وREPL والناتج.
- **الاسترداد:** تستخدم دلائل traceback المستقرة لإصلاح `pritn` من دون إعادة تثبيت أو رفع صلاحيات.
- **التحقق:** تحفظ وتلاحظ إعادة التشغيل الناجحة من المجلد المقصود.
- **الشرح:** تستطيع شرح سبب تغيّر ما يراه Python عند وجود تعديل غير محفوظ أو working directory خاطئ.

تثبت نتيجة 5/5 أنك حققت مخرج البرنامج الأول. إذا فشل معيار واحد، فأعد حلقة الحفظ والتشغيل والإصلاح وحدها؛ أدوات الإعداد الاختيارية غير مطلوبة.

### Checkpoint البيئة

داخل `.venv` المفعّلة، تأكد من أن `python --version` و`python -c "import sys; print(sys.executable)"` و`python -m pip --version` تستجيب. يجب أن يشير مسار الملف التنفيذي إلى داخل `.venv`.

امنح نفسك نقطة لكل معيار:
- **الصحة:** الإصدار 3.11 أو أحدث، وتنجح الفحوص الثلاثة.
- **الوضوح:** تسجل ملاحظتك القصيرة الأمر ومسار الملف التنفيذي اللذين استخدمتهما فعلًا.
- **الاسترداد:** تستطيع تنفيذ `deactivate` وتعرف كيف تتراجع عن أي تعديل اختياري للطرفية.
- **التحقق:** بعد إعادة تفعيل `.venv` يظل المسار داخل البيئة.
- **الشرح:** تستطيع أن تشرح بكلماتك لماذا يمنع `python -m pip` والبيئة الافتراضية خلط عمليات التثبيت.
