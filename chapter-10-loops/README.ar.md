# الفصل 10 · الحلقات والكفاءة والتكرار

[English](README.md) · [Español](README.es.md) · [Català](README.ca.md) · [Svenska](README.sv.md) · العربية (الحالية)

## ما الذي سنبنيه؟
سندرس `for` و`while` وأنماطاً مثل الترقيم والتجميع والخروج المبكر، وسنخطو أولى خطواتنا في تعقيد الزمن لفهم تكلفة تكرار العمليات. سترى أمثلة بالقوائم والقواميس والحلقات المتداخلة، إضافة إلى طرائق بسيطة لتقدير مدى «كلفة» شيفرتك.

## مسار التعلم
1. **النموذج الذهني**: الحلقة تكرار مضبوط.
2. **`for` على الكائنات القابلة للتكرار**: القوائم والنطاقات والقواميس.
3. **`while` والشروط**: التكرار حتى يتغير شيء ما.
4. **العدادات والمراكم**: أنماط احترافية.
5. **الحلقات المتداخلة**: متى تستخدمها وما تكلفتها.
6. **حدس التعقيد**: O(n) وO(n²) بأمثلة قابلة للقياس.
7. **تحسين أساسي**: الخروج المبكر واختيار بنية البيانات الصحيحة.

## أهداف التعلم
- كتابة حلقات `for` و`while` مقروءة تستطيع الاستدلال على سلوكها.
- تجميع النتائج، مثل المجاميع والقوائم الجديدة، والتوقف باستخدام `break` و`continue`.
- تحليل عدد مرات تشغيل الحلقة وتقدير رتبة نموها.
- إدراك متى تكون الحلقة المتداخلة مكلفة والبحث عن بدائل.
- التحقق من سلوك الحلقات باختبارات بسيطة.

## لماذا يهم هذا؟
تعالج الحلقات تجميعات كاملة، لكنها قد تتحول أيضاً إلى اختناقات. يساعدك فهم سلوكها على كتابة شيفرة قابلة للتوسع واكتشاف فرص التحسين قبل الإنتاج.

### مغامرة صغيرة
تشبه الحلقة التدرب على حركة رياضية: تكرر الفعل نفسه حتى يصبح سهلاً. لكنك إذا كررت «أكثر مما ينبغي» تتعب وتهدر الوقت. ولهذا سنتعلم أيضاً تكلفة الحلقات المتداخلة.

## توقّع قبل التنفيذ
قبل تشغيل المثال الأول، توقّع الأسطر الثلاثة المطبوعة وعدد مرات تنفيذ جسم الحلقة. ثم أجب: هل تتضمن `range(2, 8, 2)` القيمة `8`، وكم قيمة تنتج؟ اكتب إجاباتك أولاً، ثم شغّل الأمثلة واشرح أي اختلاف. بهذه العادة يصبح التنفيذ دليلاً بدلاً من أن يكون تخميناً.

## 1. استخدام `for` مع الكائنات القابلة للتكرار

```python runnable
tareas = ["instalar dependencias", "correr tests", "hacer deploy"]
for indice, tarea in enumerate(tareas, start=1):
    print(f"{indice}. {tarea}")
```

- تضيف `enumerate` عداداً من دون إدارته يدوياً.
- وتعمل أيضاً مع السلاسل النصية والقواميس (`for clave, valor in dic.items()`)، والمجموعات ذات الترتيب الاعتباطي، والمولّدات.

### نطاقات مضبوطة
```python runnable
for numero in range(5):  # 0 to 4
    print(numero)
```

---

## 2. استخدام `while` عندما لا تعرف عدد التكرارات

```python runnable
contador = 0
while contador < 3:
    print(f"Intento {contador}")
    contador += 1
```

- عرّف شرطاً واضحاً وحدّث المتغيرات داخل الحلقة لتجنب الحلقات اللانهائية.

### `break` و`continue`
```python runnable
for intento in range(5):
    if intento == 3:
        break  # stops the loop completely
    if intento % 2 == 0:
        continue  # jumps to the next iteration
    print(intento)
```

## 3. المراكم والتحويلات

```python runnable
numeros = [1, 2, 3, 4]
acumulado = 0
for n in numeros:
    acumulado += n
print(acumulado)
```

### إنشاء تجميعات جديدة
```python illustrative
cuadrados = []
for n in numeros:
    cuadrados.append(n**2)
```

- يفيد ذلك عندما تحتاج إلى منطق يتجاوز ما يناسب استيعاب قائمة.

## 4. الحلقات المتداخلة والتكلفة

```python runnable
datos = [[1, 2], [3, 4, 5]]
for fila in datos:
    for valor in fila:
        print(valor)
```

- إذا اشتغلت الحلقة الخارجية `n` مرة والداخلية `m` مرة، يكون مجموع التكرارات `n * m`.
- عندما تكون `n ≈ m` يصبح التعقيد O(n²). وهو مقبول للجداول الصغيرة ومكلف عندما تكبر.

### مثال مع تحقق
```python runnable
usuarios = ["noor", "frej", "taha"]
permisos = ["ver", "editar", "borrar"]
combinaciones = []
for usuario in usuarios:
    for permiso in permisos:
        combinaciones.append((usuario, permiso))
print(len(combinaciones))  # 9, cartesian product
```

## 5. تقدير التعقيد حدسياً

| النمط | التكرارات | الرتبة التقريبية |
| --- | --- | --- |
| المرور على قائمة مرة واحدة | n | O(n) |
| حلقتان متتابعتان | n + m | O(n + m) |
| حلقتان متداخلتان (n * m) | n * m | O(n·m) |
| بحث خطي | n | O(n) |

- تعني O(n) أن الزمن ينمو متناسباً مع حجم الإدخال.
- تنمو O(n²) أسرع بكثير؛ فمضاعفة n تعني نحو أربعة أضعاف التكرارات.

### القياس باستخدام `time.perf_counter()`
```python runnable
import time

datos = list(range(100000))
start = time.perf_counter()
suma = 0
for valor in datos:
    suma += valor
end = time.perf_counter()
print(f"Loop O(n) took {end - start:.4f}s")
```

## 6. تحسين أساسي
- تجنب الحلقات المتداخلة عندما توجد بنية أسرع؛ فالعضوية في `set` ذات تعقيد O(1) في المتوسط.
- استخدم `break` بمجرد العثور على ما تحتاج إليه.
- انقل العمل الثابت إلى خارج الحلقة.

### مثال: بحث فعال
```python runnable
def contiene(lista, objetivo):
    for elemento in lista:
        if elemento == objetivo:
            return True
    return False
```

- التعقيد O(n). ويمكنك تقليله إلى O(1) في المتوسط باستخدام مجموعة.

### 10-0 · مسح محدود

توقع القيم المقبولة والمجموع، ثم شغّل السلوك العادي والحدي في حلقة محدودة:

```python runnable
values = [3, -1, 4, 0, 9]
accepted = []
total = 0

for value in values:
    if value < 0:
        continue
    if value == 0:
        break
    accepted.append(value)
    total += value

print(accepted)
print(total)
```

تحاول الكتلة التالية عمداً تكرار عدد صحيح؛ `TypeError` هو التشخيص المتوقع:

<!-- bookcheck: expect-error="TypeError" -->
```python expected-error
for item in 3:
    print(item)
```

تعافَ بتوفير كائن قابل للتكرار وتحقق من التكرارات الثلاثة المحدودة:

```python runnable
for item in range(3):
    print(item)
```

سجل المخرجات العادية وحد القائمة الفارغة ومخرجات التعافي. توقف هنا في المسار الأساسي.

## أخطاء شائعة
- نسيان تحديث العدادات في `while` ← يؤدي إلى حلقات لانهائية.
- تعديل القائمة نفسها التي تكرر عليها ← يؤدي إلى تخطي عناصر؛ استخدم نسخة.
- تداخل الحلقات من دون تقدير الأحجام ← يؤدي إلى انفجار زمن التشغيل.
- تنفيذ عمل ثقيل داخل الحلقة مع إمكان تنفيذه مرة واحدة خارجها.

## حلول مشروحة
1. **عداد حروف العلة**: استخدم حلقة `for` واحدة لكل محرف؛ ويزيد قاموس العدادات (`vocales[letra] = vocales.get(letra, 0) + 1`). التعقيد O(n).
2. **جدول الضرب**: حلقتا `for` من عشرة تكرارات لكل منهما ← 100 تكرار، أو O(n²) إذا نمت n. تتجاوز `continue` طباعة النتائج الصغيرة.
3. **البحث المبكر**: استخدم `for usuario in usuarios` ثم `if usuario == nombre: return True`؛ وإذا انتهت الحلقة فأعد `False`. يغطي الاختبار المسارين.

## نقطة تحقق وسلّم تقييم
كيّف 10-0 كجمل مباشرة لثلاث قوائم منفصلة: بلا صفر، وصفر في البداية، وسوالب قبل الصفر. اطبع القيم المقبولة والمجموع لكل تشغيل، ثم أعد إنتاج `TypeError` للكائن غير القابل للتكرار وصححه. لا تضع الكود في دالة ولا اختبار pytest.

امنح نقطة لكل معيار: **الصحة** (تنجح الحالات الثلاث)، و**المقروئية** (أسماء واضحة وحلقة مركزة)، و**تدفق التحكم** (تتبع `continue` و`break` القواعد)، و**التعافي** (يتبع الخطأ المتوقع كود عامل محدود)، و**الدليل** (تسجل التوقعات والمخرجات). تعني 4/5 أنك تستطيع المتابعة؛ وإلا فراجع الأقسام 1–3. التعقيد والقياس والدوال وpytest أدلة اختيارية.
