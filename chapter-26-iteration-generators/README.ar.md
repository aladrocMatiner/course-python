# الفصل 26 · التكرار والمكرّرات والمولّدات

[English](README.md) · [Español](README.es.md) · [Català](README.ca.md) · [Svenska](README.sv.md) · العربية

## ما الذي سنبنيه

تستقبل برامج كثيرة سلسلة من القيم ثم تحوّلها: تتحول الأسماء إلى تسميات،
والقراءات إلى ملخصات، والمجموعات إلى تدفق واحد. أنت تعرف بالفعل كيف تنفذ ذلك
بحلقة `for`. يحافظ هذا الفصل على هذا النموذج الذهني الموثوق، ثم يضيف ثلاث
أدوات بترتيب مقصود:

1. **صيغ الاستيعاب و`enumerate` و`zip`** لإجراء تحويلات صغيرة ومقروءة على
   المجموعات.
2. **المكرّرات** لفهم موضع حالة الاجتياز وسبب إمكان نفاد تدفق البيانات.
3. **المولّدات** لإنتاج القيم عند الطلب مع إبقاء المستهلك محدودًا بوضوح.

المشروع الذي سنطوره هو خط معالجة بالتكرار للوحة نتائج صغيرة واصطناعية لورشة
عمل. مرجعه التنفيذي هو
[`examples/iteration_pipeline.py`](examples/iteration_pipeline.py)، وتدعمه
اختبارات من المكتبة القياسية. لا يستخدم شبكة أو بيانات اعتماد أو بيانات
شخصية أو ملفات أو خيوط تنفيذ أو حزمًا خارجية.

## أهداف التعلم

عند إكمال المسار الذي تختاره، ستتمكن من:

- **O1 — التحويل بوضوح:** اشتقاق صيغة استيعاب لقائمة أو قاموس أو مجموعة من
  حلقة عادية، واختيار الصيغة الأوضح.
- **O2 — الترقيم والإقران بأمان:** استخدام `enumerate` لمواضع العرض،
  والاختيار المقصود بين اقتطاع `zip` و`zip(..., strict=True)`.
- **O3 — شرح حالة المكرّر:** التمييز بين الكائن القابل للتكرار والمكرّر،
  واستخدام `iter` و`next`، وملاحظة النفاد، وإعادة إنشاء اجتياز عندما يسمح
  المصدر بذلك.
- **O4 — حماية الاجتياز:** استخدام قيمة افتراضية غير ملتبسة عند النفاد،
  وتجنب تغيير بنية المجموعة الجاري اجتيازها.
- **O5 — بناء عمل مؤجل ومحدود:** شرح تعبيرات المولّد و`yield` والاستهلاك
  لمرة واحدة و`yield from` وحدّ صريح ومنتهٍ.
- **O6 — التعافي والتنظيف:** تحديد الخطأ المؤجل عند خطوة الاستهلاك، وإنهاء
  المولّد باستخدام `return`، وتنفيذ تنظيف حتمي بعد الإغلاق المبكر.

يربط [سجل التتبّع](TRACEABILITY.md) كل هدف بالشرح والتدريب والتعافي والحل
ونقطة التحقق والمصدر المرافق والاختبارات.

نقاط التحقق لتأكيد أي فجوات.

## مسرد صغير

- **قابل للتكرار (Iterable):** كائن يستطيع توفير مكرّر. القوائم والصفوف
  والقواميس والمجموعات والسلاسل النصية أمثلة مألوفة.
- **المكرّر (Iterator):** كائن ذو حالة ينتج قيمة تالية واحدة في كل مرة،
  ويتذكر إلى أي موضع وصل الاجتياز.
- **النفاد (Exhaustion):** الحالة التي لا تبقى فيها للمكرّر قيمة تالية.
  عندها يرفع `next` المباشر الاستثناء `StopIteration` ما لم تُعطَ قيمة
  افتراضية.
- **صيغة الاستيعاب (Comprehension):** صياغة مختصرة لبناء مجموعة ملموسة من
  كائن قابل للتكرار.
- **المولّد (Generator):** مكرّر ينشئه تعبير مولّد أو دالة تحتوي على `yield`.
- **التنفيذ المؤجل (Lazy):** إنتاج العمل عندما تُطلب القيم. لا يعني التأجيل
  أن العمل منتهٍ أو سريع أو آمن تلقائيًا.
- **المستهلك (Consumer):** الحلقة أو `next` أو `list` أو أي عملية أخرى تطلب
  قيمًا من مكرّر.

التسلسل القابل للملاحظة واحد دائمًا: احصل على مكرّر، واطلب قيمة، ثم استلم
قيمة أو إشارة نفاد، وبعدها توقف أو اطلب من جديد. هذا التسلسل النصي هو البديل
الكتابي لأي مخطط لحالة المكرّر قد ترسمه؛ فلا يعتمد أي معنى على اتجاه سهم أو
لون.

## المسار الأساسي: تحويلات مقروءة

### E1. اشتق صيغة استيعاب من حلقة

الهدف: بناء مجموعة صغيرة من دون فقدان النموذج الذهني للحلقة العادية.

لنفترض أن ورشة تسجل ثلاث درجات اصطناعية. قبل تشغيل المثال، توقّع قيمة
`doubled` بعد كل دورة من الحلقة.

```python runnable
scores = [3, 5, 8]
doubled = []

for score in scores:
    doubled.append(score * 2)

print(doubled)
```

```text output
[6, 10, 16]
```

اقرأ الحلقة بهذا الترتيب: خذ قيمة `score`، واحسب `score * 2`، وأضف النتيجة،
ثم كرر. تعبّر صيغة استيعاب القائمة عن التحويل نفسه مع وضع تعبير النتيجة أولًا:

```python runnable
scores = [3, 5, 8]
doubled = [score * 2 for score in scores]
print(doubled)
```

```text output
[6, 10, 16]
```

تعني الأقواس المربعة «ابنِ قائمة». وداخلها:

1. `score * 2` هو القيمة التي ستُخزّن؛
2. `score` هو اسم الهدف الحالي؛
3. `scores` هو الكائن القابل للتكرار الذي يزوّد القيم.

يمكن لشرط واحد أن يرشّح القيم، لكنه يأتي بعد الكائن القابل للتكرار. توقّع
أي الدرجات ستبقى قبل التشغيل:

```python runnable
scores = [3, 5, 8]
large_doubles = [score * 2 for score in scores if score >= 5]
print(large_doubles)
```

```text output
[10, 16]
```

تستخدم صيغ استيعاب القواميس والمجموعات ترتيب القراءة نفسه. يخزن القاموس
أزواج مفتاح وقيمة، وتزيل المجموعة القيم المتساوية المكررة. نرتب المجموعة
فقط للحصول على عرض مستقر؛ فترتيب اجتياز المجموعة ليس عقد صحة.

```python runnable
names = ["Noor", "Frej", "Taha"]
length_by_name = {name: len(name) for name in names}
unique_lengths = {len(name) for name in names}

print(length_by_name)
print(sorted(unique_lengths))
```

```text output
{'Noor': 4, 'Frej': 4, 'Taha': 4}
[4]
```

حالة الإدخال الفارغ هادئة ومفيدة: لا يُنفَّذ أي جسم، فتنتج كل صيغة استيعاب
مجموعة فارغة من نوعها.

```python runnable
print([value for value in []])
print({value: len(value) for value in []})
print({value for value in []})
```

```text output
[]
{}
set()
```

**عدّل — TODO للهدف O1:** أعد كتابة هذه الحلقة في صيغة استيعاب قائمة واحدة
مقروءة. أبقِ تحويلًا واحدًا وشرطًا واحدًا بالضبط.

```python todo
scores = [2, 4, 7, 9]
selected = []
for score in scores:
    if score >= 5:
        selected.append(score + 1)

# TODO: replace the loop with selected = [...]
print(selected)
```

**تلميح:** اقرأ الجسم الأصلي بالترتيب: خزّن `score + 1`، وخذ كل `score` من
`scores`، واحتفظ به عندما يتحقق `score >= 5`.

**الحل المشروح:** يأتي تعبير النتيجة أولًا، ثم الحلقة نفسها، ثم الشرط نفسه.
النتيجة العادية هي `[8, 10]`، وحتى إذا كانت `scores` فارغة فستكون النتيجة
`[]`.

```python runnable
scores = [2, 4, 7, 9]
selected = [score + 1 for score in scores if score >= 5]
print(selected)
```

```text output
[8, 10]
```

خطأ شائع: ضغط حلقات متداخلة وشروط متعددة وتأثيرات جانبية وتعيينات في تعبير
واحد. تكون صيغة الاستيعاب مفيدة عندما تكون أسهل قراءة. احتفظ بالحلقة العادية
عندما تروي الخطوات بهدوء أوضح.

### E2. رقّم المواضع من دون عدّاد يدوي

الهدف: إرفاق مواضع موجهة للقارئ من دون تحديث متغير منفصل.

ينتج `enumerate` أزواجًا من `(position, value)`. يفيد `start=1` في التسميات
المعروضة لشخص، لكنه لا يغيّر فهرسة المجموعة التي تبدأ من الصفر. توقّع السطرين
المطبوعين:

```python runnable
names = ["Noor", "Frej"]

for position, name in enumerate(names, start=1):
    print(f"{position}: {name}")
```

```text output
1: Noor
2: Frej
```

الحالة الحدية هي كائن قابل للتكرار فارغ: يُنفذ جسم الحلقة صفر مرة ولا يطبع
شيئًا. لا يوجد موضع خاص ينبغي اختراعه.

**عدّل — TODO للهدف O2:** غيّر رقم بداية العرض إلى `10`، ثم توقّع التسميتين
قبل التشغيل.

```python todo
names = ["Noor", "Frej"]
# TODO: enumerate names starting at 10 and print "position: name"
```

**تلميح:** غيّر وسيط `start` فقط؛ لا تضف عدّادًا منفصلًا ثم تزده.

**الحل المشروح:** ينتج `enumerate(names, start=10)` القيمتين
`(10, "Noor")` و`(11, "Frej")`. لا تتغير القائمة نفسها.

```python runnable
names = ["Noor", "Frej"]
print(list(enumerate(names, start=10)))
```

```text output
[(10, 'Noor'), (11, 'Frej')]
```

### E3. اقرن البيانات من دون فقدان صامت

الهدف: إظهار عقد الطول عندما ينتمي مصدرا بيانات إلى بعضهما.

يتوقف `zip` العادي عند نفاد الإدخال الأقصر. قد يكون ذلك مقصودًا، لكنه قد يخفي
أيضًا درجة مفقودة:

```python runnable
names = ["Noor", "Frej"]
scores = [7]
print(list(zip(names, scores)))
```

```text output
[('Noor', 7)]
```

تختفي `"Frej"` لأن الكائن القابل للتكرار للدرجات لا يحتوي على قيمة ثانية.
عندما تكون الأطوال المتساوية جزءًا من الصحة، اطلب ذلك العقد صراحة باستخدام
`strict=True`. تستهلك الدالة المرافقة كائن `zip` داخل قائمة، ولذلك لا يمكن أن
يبقى عدم التطابق مخفيًا في كائن `zip` غير مستهلك.

<!-- bookcheck: path="chapter-26-iteration-generators/examples/iteration_pipeline.py" check="learning:contract" -->
```python source-ref
def strict_pairs(left, right):
    """Return pairs, rejecting unequal input lengths."""
    return list(zip(left, right, strict=True))
```

في المسار الصحيح، تتحقق الاختبارات من أن
`strict_pairs(["Noor", "Frej"], [7, 9]) == [("Noor", 7), ("Frej", 9)]`.
وينتج إدخالان فارغان القيمة الصحيحة `[]`.

توقّع متى يفشل المثال التالي: عند إنشاء `zip` أم عندما تطلب منه `list` القيم؟

<!-- bookcheck: expect-error="ValueError" -->
```python expected-error
names = ["Noor", "Frej"]
scores = [7]
pairs = zip(names, scores, strict=True)
print(list(pairs))
```

الإشارة المستقرة هي `ValueError`. لا يفعل الإنشاء أكثر من تكوين المكرّر؛
ويكتشف الاستهلاك أن الإدخال الأيمن انتهى أولًا. قد يتغير النص الكامل للتتبّع
بين إصدارات صيانة Python.

**التعافي:** صحح الدرجة المفقودة، ثم أعد تشغيل العقد الصارم نفسه.

```python runnable
names = ["Noor", "Frej"]
scores = [7, 9]
print(list(zip(names, scores, strict=True)))
```

```text output
[('Noor', 7), ('Frej', 9)]
```

تكون إزالة `strict=True` تصميمًا صالحًا أيضًا، ولكن فقط عندما يكون تجاهل
القيم الزائدة في النهاية مقبولًا صراحة. دوّن ذلك القرار في عقد المجال، ولا
تجعل الاقتطاع الصامت إصلاحًا عرضيًا.

### التحدي الأساسي الموجّه

ابنِ لوحة نتائج مرقمة. توقّع أولًا سطري الخرج والخطأ الناتج من درجة مفقودة.

```python todo
names = ["Noor", "Frej"]
scores = [7, 9]

# TODO 1: pair names and scores with strict=True.
# TODO 2: enumerate the pairs starting at 1.
# TODO 3: build lines like "1. Noor: 7" with a list comprehension.
# TODO 4: print each line.
```

**تلميح:** أنشئ `pairs` أولًا. بعد ذلك تستطيع صيغة الاستيعاب فك
`position, (name, score)` من `enumerate(pairs, start=1)`.

**الحل المشروح:** يحمي الإقران الصارم محاذاة البيانات، وينشئ الترقيم مواضع
العرض، وتنسّق صيغة الاستيعاب سطرًا واحدًا لكل زوج. لكل خطوة مسؤولية واحدة.

```python runnable
names = ["Noor", "Frej"]
scores = [7, 9]
pairs = list(zip(names, scores, strict=True))
lines = [
    f"{position}. {name}: {score}"
    for position, (name, score) in enumerate(pairs, start=1)
]

for line in lines:
    print(line)
```

```text output
1. Noor: 7
2. Frej: 9
```

عندما تكون `names = []` و`scores = []`، تكون `pairs` و`lines` فارغتين ولا
يُطبع شيء. وعند اختلاف الطولين، يكون `ValueError` هو التشخيص المتوقع؛ أصلح
البيانات بدل إخفاء عدم التطابق.

### نقطة التحقق الأساسية ومعايير تقييمها

أكمل لوحة النتائج للإدخالات العادية والفارغة وغير المتساوية. اشرح سبب ترتيب
الخرج العادي، وسبب صحة النتيجة الفارغة، وسبب ظهور خطأ عدم التطابق أثناء
الاستهلاك.

أضف نقطة واحدة لكل معيار:

- **الصحة:** ينتج الإدخال العادي سطري الترقيم المحددين بالضبط.
- **الحالة الحدية:** ينتج إدخالان فارغان نتيجة فارغة من دون صف مختلق.
- **التعافي:** ترفع الإدخالات غير المتساوية `ValueError`، وينجح الإدخال
  المصحح.
- **سهولة القراءة:** تحتوي كل صيغة استيعاب على تحويل واحد واضح وشرط بسيط
  واحد على الأكثر.
- **الشرح:** تستطيع التمييز بين مواضع العرض وفهارس المجموعة وعقد الإقران
  الصارم.

تكمل نتيجة 4/5، على أن تشمل **التعافي**، المسار الأساسي. يمكنك التوقف بأمان
والانتقال إلى الفصل 12. تأمل: أين يمكن أن يكون الاقتطاع الصامت خطيرًا في
برنامج تعرفه؟

## المسار الاحترافي: حالة مكرّر تستطيع شرحها

### P1. الكائن القابل للتكرار والمكرّر يؤديان دورين مختلفين

الهدف: تحديد موضع حالة الاجتياز بدل تخيل أن حلقة `for` تتذكر موضعًا بطريقة
سحرية.

القائمة **كائن قابل للتكرار**: يمكنها توفير مكرّر. يطلب `iter(values)` واحدًا.
يخزن المكرّر الناتج تقدم الاجتياز، ويطلب `next(cursor)` قيمة واحدة من ذلك
المكرّر بعينه.

توقّع كل سطر قبل التشغيل:

```python runnable
values = ["A", "B"]
cursor = iter(values)

print(iter(cursor) is cursor)
print(next(cursor))
print(next(cursor))
print(next(cursor, "done"))
```

```text output
True
A
B
done
```

التسلسل بالنثر هو:

1. تستطيع `values` إنشاء عمليات اجتياز.
2. يبدأ `cursor` قبل القيمة الأولى.

4. يتقدم الثاني ويعيد `"B"`؛

تكون `iter(cursor) is cursor` هي `True` لأن طلب مكرّر من مكرّر يعيد الكائن

```python source-ref
def countdown(start):
    """Yield ``start`` down to 1 after validating a finite bound."""

        current -= 1
```

### A3. ضع حدًا لمصدر غير منتهٍ لولا ذلك

### A6. تنظيف حتمي بعد الإغلاق المبكر

الهدف: تحرير مورد محلي مملوك عندما يكتمل مولّد بدأ العمل أو يفشل أو يُغلق

يقبل هذا المصدر المرافق دالة `close` وهمية قابلة للاستدعاء كي يستطيع الاختبار

<!-- bookcheck: path="chapter-26-iteration-generators/examples/iteration_pipeline.py" check="learning:contract" -->
```python source-ref
def managed_values(values, close):
    """Yield values and call ``close`` once when an active generator ends."""
    if not callable(close):
        raise TypeError("close must be callable")
    try:
        yield from values
    finally:
        close()
```

يبدأ اختبار التنظيف المولّد، ويستهلك `"A"`، ويستدعي `cursor.close()`، ثم
يلاحظ `events == ["closed"]`. لا يضيف استدعاء `close` مرة أخرى حدثًا ثانيًا.
كما يشغّل النفاد الكامل الطبيعي التنظيف مرة واحدة. وترفع قيمة تنظيف غير قابلة
للاستدعاء `TypeError` عند بدء الاستهلاك.

حالة حدية مهمة: لا يؤدي إغلاق مولّد لم يبدأ قط إلى دخول جسمه، ولذلك لا يمكنه
الاعتماد على ذلك الجسم لاقتناء مورد ثم تحريره. اقتنِ المورد فقط داخل فترة
التشغيل الفعلية التي تتحكم فيها، وأغلق صراحة مولّدًا بدأ تشغيله عندما يتوقف
المستهلك مبكرًا. لا تعتمد على توقيت جمع المهملات.

**عدّل — TODO للهدف O6:** استخدم `events = []`، وأنشئ مولّد

```text output
[0, 1, 4, 9]
```

تعيد الحالة الحدية صفر `[]`. يحتاج الحد السالب أو الضخم الذي يختاره المتعلم
إلى تحقق قبل أن يصل إلى هذا البناء؛ ويملك `bounded_squares` في المصدر المرافق
ذلك العقد المتحقق منه.

### نقطة التحقق المتقدمة ومعايير تقييمها

أكمل التقرير المحدود، والتعافي من الخطأ المؤجل، ومهمة TODO للتنظيف. ثم اشرح
متى يبدأ كل مولّد عمله وكيف ينتهي.

أضف نقطة واحدة لكل معيار:

- **التأجيل:** تميّز الإنشاء من أول خطوة استهلاك.
- **الحدود:** لكل مصدر غير منتهٍ لولا ذلك حد مستهلك صريح ومتحقق منه ومنتهٍ.
- **حالة الاستخدام مرة واحدة:** يُعاد إنشاء المولّد النافد أو الفاشل عمدًا.
- **التعافي:** يُصحح الإدخال غير الصالح وينجح المثال المجاور بعد إصلاحه.
- **التفويض:** يحافظ `yield from` على ترتيب المجموعات المنتهية، ومنها
  المجموعات الفارغة.
- **التنظيف والشرح:** يغلق المولّد الذي بدأ تشغيله مورده الوهمي مرة واحدة
  بالضبط، وتشرح سبب كون `return` نهاية طبيعية.

تكمل نتيجة 5/6، على أن تشمل **الحدود** و**التنظيف والشرح**، المسار المتقدم
الاختياري. تأمل: أي حد يخص المنتج، وأي حد يخص المستهلك، ومن يملك التنظيف في
خط المعالجة؟

## أخطاء شائعة وطرق تعافٍ هادئة

- **صيغ استيعاب كثيفة:** إذا احتجت إلى فك السطر مرتين، فعد إلى حلقة مسماة أو
  افصل القيم الوسيطة.
- **الاعتماد على ترتيب المجموعة:** قارن انتماء العناصر إلى المجموعة، أو
  رتّبها فقط للحصول على عرض مستقر؛ ولا تعد بترتيب عرض اعتباطي.
- **استخدام `zip` العادي عندما يجب أن تتطابق الأطوال:** اختر `strict=True`،
  واستهلك النتيجة، وصحح عدم تطابق المصدر، ثم أعد التشغيل.
- **افتراض أن `zip` يتحقق عند الإنشاء:** يكتشف نفاد الإدخال مع تقدم المستهلك.
- **استدعاء `next` إلى الأبد:** تعامل مع النفاد بحلقة `for` أو قيمة افتراضية
  مقصودة أو مكرّر جديد من مصدر قابل لإعادة الاستخدام.
- **معاملة المولّد كأنه يقبل الإرجاع إلى البداية:** أعد إنشاءه من الإدخالات
  الأصلية.
- **تغيير المجموعة الجاري اجتيازها:** ابنِ نتيجة منفصلة أو لقطة صريحة.
- **وصف خط معالجة بأنه «آمن لأنه مؤجل»:** أضف حدًا للعناصر أو الوقت أو
  الإدخال، وتنظيفًا حتميًا للموارد.
- **توقع التحقق عند إنشاء المولّد:** تذكر أن جسم المولّد يبدأ عند الاستهلاك؛
  اختبر الطلب الأول.
- **رفع `StopIteration` داخل مولّد:** استخدم `return` للنهاية الطبيعية.

الأخطاء في هذا الفصل ملاحظات وليست إخفاقات شخصية. اختصر المثال إلى منتج واحد
ومستهلك واحد وقيمة تالية واحدة متوقعة؛ ثم أصلح أصغر عقد وأعد تشغيل المثال
الصحيح المجاور له.

## التحقق من العقد القابل للتنفيذ

شغّل حزمة اختبارات المكتبة القياسية المعلنة من جذر المستودع:

```bash illustrative
python -B -m unittest discover -s chapter-26-iteration-generators/examples/tests -t chapter-26-iteration-generators/examples -p 'test_*.py'
```

يمنع `-B` إنشاء ذاكرة bytecode مؤقتة. يضيف الاكتشاف دليل `examples` المرافق
بوصفه جذر استيراد الاختبارات، ولا يشغّل سوى `test_*.py` تحت حزمة `tests`.
تتحقق الحزمة من:

- الإقران الصارم العادي والفارغ، إضافة إلى `ValueError` للأطوال غير المتساوية؛
- العد التنازلي العادي والصفر والحدود غير الصالحة والنفاد وإعادة الإنشاء؛
- أول خمسة مربعات محدودة والحدود غير الصالحة؛
- التفويض المنتهي مع مجموعات فارغة؛
- `ZeroDivisionError` المؤجل والتعافي بإدخال مصحح؛
- الإغلاق الجزئي والنفاد الطبيعي والتنظيف غير الصالح والتنظيف مرة واحدة
  بالضبط.

من المتوقع أن تنتهي التعليمة برمز خروج صفر. لا يُعد غياب المفسر أو نتيجة
اختبار غير صفرية نجاحًا: اقرأ أول اختبار فاشل، وصحح أصغر عقد، وأزل الحالة
المؤقتة، ثم أعد تشغيل تلك الحالة ومثالها الصحيح المجاور.

سُجلت كتل Markdown من نوع `source-ref` تحت `learning:contract`. إنها روابط
إثبات، وليست شيفرة ينفذها مسار Markdown العام. تملك آلية التحقق الصريحة
`learning-bridges` سلوك المصدر المرافق؛ وتظل كتل `runnable` و`expected-error`
الصغيرة مؤهلة للمتحقق العام المحدود.

يثبت التحقق السلوك الذي اختُبر على المفسر المحدد. ولا يثبت وحده جودة التعليم
أو سلاسة الترجمة أو التوافق الواسع بين المنصات أو إمكانية الوصول أو الموافقة
على النشر.

## الخلاصة والتأمل الختامي

يمكنك الآن اختيار أداة الاجتياز وفق العقد:

- استخدم حلقة عادية عندما تكون خطواتها الأوضح؛
- استخدم صيغة استيعاب بسيطة لبناء مجموعة صغيرة ملموسة؛
- استخدم `enumerate` للمواضع و`zip` الصارم للإقران متساوي الطول؛
- تعامل صراحة مع حالة المكرّر والنفاد؛
- لا تستخدم مولّدًا إلا مع حد مقصود للمستهلك ومالك واضح للتنظيف.

قبل الانتقال، اشرح خط معالجة واحدًا بصوت مسموع: من أين تأتي القيم، وأين تعيش
الحالة، وما الذي يوقف الاستهلاك، وأي خطأ يمكن التعافي منه، ومن ينظف. إذا كانت
كل إجابة محددة، فلم تعد الصياغة سحرًا؛ بل أنت تصمم تدفق البيانات.
