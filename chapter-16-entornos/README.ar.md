# الفصل 16 · البيئات والتبعيات والمشاريع القابلة لإعادة الإنتاج

[English](README.md) · [Español](README.es.md) · [Català](README.ca.md) · [Svenska](README.sv.md) · العربية (الحالية)

## ما الذي سنبنيه؟
ستُعد بيئات افتراضية (`venv`)، وتثبت التبعيات باستخدام `pip`، وتميّز بين `pyproject.toml` والمتطلبات والقيود واللقطات وملفات القفل، وتتعلم تحميل متغيرات البيئة للإعداد الآمن. وسنتدرب بمشروع صغير يثبت `requests` ويستخدم ملف `.env`.

## مسار التعلم
1. **لماذا نعزل التبعيات؟**
2. **إنشاء `venv` وتنشيطها**.
3. **تثبيت الحزم باستخدام `pip`**.
4. **تسجيل التبعيات بدقة**: التصريحات والإصدارات المثبتة والقيود واللقطات والأقفال.
5. **أساسيات `pyproject.toml`**.
6. **متغيرات البيئة (`os.environ`) و`.env`**.

## أهداف التعلم
- إنشاء البيئات الافتراضية وتنشيطها على Windows وmacOS وLinux.
- تثبيت المكتبات وتسجيل التبعيات المباشرة عن قصد.
- استخدام `pip freeze` كلقطة خاصة بالمفسّر والمنصة، لا كمحلّل تبعيات أو ملف قفل.
- شرح الدليل الأقوى الذي يقدمه قفل محلول.
- تحميل الإعدادات الحساسة من متغيرات البيئة.

## لماذا يهم هذا؟
من دون بيئات معزولة، يمكن لمشروع أن يعطّل مشروعاً آخر. والتبعيات المضبوطة هي أساس العمل الجماعي الاحترافي.

### مغامرة صغيرة
فكّر في كل بيئة افتراضية كصندوق LEGO يحتوي على القطع الدقيقة لمشروع واحد. إذا مزجت قطع المجموعات كلها، يصبح بناء أي شيء فوضى. باستخدام `venv` تُبقي كل مجموعة منفصلة، وتستطيع دائماً إعادة بناء النموذج من دون فقدان القطع.

## المتطلبات المسبقة
- أوامر الطرفية الأساسية والوحدات من الفصل 15.
- CPython 3.11+ مع `venv` و`pip`؛ يتطلب تثبيت الحزم اتصالاً بالشبكة، لكن التدريب على متغيرات البيئة محلي.

## توقّع قبل التشغيل
قبل إنشاء البيئة، توقّع المفسر الذي سيستهدفه `python -m pip` قبل التنشيط وبعده. تحقق من المسارين، ثم اشرح لماذا يعد المسار غير المتوقع مشكلة في الإعداد، لا خللًا في الحزمة.

## 1. إنشاء `venv` وتنشيطها

```bash illustrative
python -m venv .venv
# macOS/Linux
source .venv/bin/activate
# Windows PowerShell
.\.venv\Scripts\Activate.ps1
```

- ستظهر `(.venv)` في طرفيتك لتبين أنك داخل البيئة.
- استخدم `deactivate` للخروج.

إذا التبس عليك أي `pip` تستخدم، فهذه الحيلة تعمل دائماً:
```bash illustrative
python -m pip install requests
```
وهي تضمن التثبيت في بايثون الذي تشغله.

---

## 2. تثبيت الحزم

```bash illustrative
python -m pip install requests
python -c "import requests; print(requests.__version__)"
```

- لكل بيئة نسختها الخاصة من `pip`.

### `requirements.txt`
```bash illustrative
python -m pip freeze > requirements.txt
git add requirements.txt
```

- لإعادة إنشاء اللقطة في بيئة نظيفة أخرى: `python -m pip install -r requirements.txt`.
- يعرض `pip freeze` ما هو مثبت بصيغة ملف المتطلبات. ولا يحل بيئة جديدة ولا ينشئ قفلاً محكماً، وقد تختلف النتيجة بين إصدارات Python وأنظمة التشغيل.

### خمسة سجلات بوظائف مختلفة
- **تصريح المشروع:** يعبّر `[project].dependencies` عن متطلبات التشغيل المباشرة، غالباً في صورة نطاقات متوافقة. ولا يسجل الرسم البياني المحلول كاملاً.
- **إصدار مباشر مثبت:** يثبت `requests==X.Y.Z` إصداراً مطلوباً واحداً، لكنه لا يوضح وحده جميع التبعيات المتعدية.
- **قيد:** يحد ملف القيود الإصدارات إذا احتاج إليها متطلب آخر؛ ولا يؤدي الإدخال إلى التثبيت. مثال: `python -m pip install -c constraints.txt requests`.
- **لقطة البيئة:** يلتقط `pip freeze` الحزم المثبتة في المفسّر والمنصة الحاليين بصيغة المتطلبات. وهذا دليل مفيد، لكنه ليس نتيجة محلّل تبعيات ولا قفلاً متعدد المنصات.
- **قفل محلول:** تسجل أداة القفل الحل الكامل ونطاق صلاحيته، وغالباً مع قيم hash وعلامات البيئة. تحقق من الأداة ومصفوفة الهدف بالضبط قبل ادعاء قابلية إعادة الإنتاج؛ دعم `pip lock` الحالي تجريبي، ومخرجاته مقيدة بإصدار Python والمنصة الحاليين.

## 3. تصريح التبعيات في `pyproject.toml` (اختياري لكنه حديث)

```toml illustrative
[project]
name = "mi-proyecto"
version = "0.1.0"
dependencies = [
  "requests>=2.31",
]
```

- يصرّح هذا بمتطلب مباشر متوافق في بيانات الحزمة الوصفية. وليس رسماً بيانياً مجمداً للتبعيات المتعدية. ينتمي التحقق من البناء والاستيراد من دليل آخر إلى مثال `src/<package>` القابل للتثبيت في الفصل 15.

## 4. متغيرات البيئة

```python runnable
import os
API_KEY = os.environ.get("API_KEY", "dummy")
```

- لا تودع الأسرار في مستودعك.

### ملف `.env` باستخدام `python-dotenv`
```bash illustrative
python -m pip install python-dotenv
```

```python illustrative
from dotenv import load_dotenv
load_dotenv()
API_KEY = os.environ["API_KEY"]
```

- أنشئ ملف `.env` يحتوي على `API_KEY=value` وأضفه إلى `.gitignore`.

ملف `.gitignore` نموذجي:
```gitignore illustrative
.venv/
.env
__pycache__/
```

## حلول مشروحة
1. **إعداد البيئة**: يعزل `python -m venv .venv` المشروع، ويسجل `python -m pip freeze > requirements.txt` لقطة من تلك البيئة. أعد تثبيتها في بيئة نظيفة بإصدار Python والمنصة المعلنين، وتحقق من الاستيرادات؛ لا توسّع هذا الدليل إلى ادعاء محكم أو متعدد المنصات.
2. **السكربت المضبوط بالإعدادات**: تتيح `load_dotenv()` لـ`os.environ` قراءة المتغيرات من الملفات المحلية.
3. **pyproject**: وثّق التبعية المباشرة وتعليمات التثبيت، ثم وضح أن تجميد الرسم البياني الكامل يحتاج إلى قفل منفصل يولّده محلّل تبعيات.
